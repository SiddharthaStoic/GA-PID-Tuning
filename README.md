📄 Детальний опис кожного файлу та папки
📁 Папка data/ (Сховище даних)
Тут зберігатимуться всі файли логів. Вона розділена на два рівні, щоб не змішувати «брудні» дані з чистими.

raw/ — сюди ви просто зберігаєте оригінальні CSV-файли (наприклад, test_01_thrust.csv, test_PID_v1_run1.csv). Ці файли ніколи не редагуються вручную.

processed/ — сюди ваші скрипти будуть зберігати очищені дані або фінальну базу результатів випробувань - база даних SQLite (зведені таблиці, де записано: який PID, яка версія фітнес-функції та яка оцінка).

📁 Папка notebooks/ (Ваша інтерактивна лабораторія)
Тут знаходяться файли Jupyter Notebook (.ipynb). Вони потрібні для досліджень, швидкого написання коду та малювання графіків.

01_data_identification.ipynb (Первинна ідентифікація):

Що всередині: Код, який зчитує перші тести Юрія. Тут ви будуєте графіки для пошуку мінімальної та робочої тяги двигунів, а також вираховуєте балансний вектор (нормальне горизонтальне положення дрону).

02_fitness_development.ipynb (Розробка фітнес-функції):

Що всередині: У цьому блокноті ви експериментуєте з формулою оцінки. Ви берете лог невдалого польоту і лог вдалого, будуєте графіки перерегулювання (Overshoot) та часу заспокоєння (Settling time). Тут ви підбираєте вагові коефіцієнти для формули фітнес-функції.

03_genetic_algorithm.ipynb (Запуск та моніторинг еволюції):

Що всередині: Головний «пульт керування». Тут імпортуються готові модулі з папки src, запускається генетичний алгоритм, формуються JSON-команди з новими PID для відправки на стенд та візуалізується прогрес покращення оцінки від покоління до покоління.4


📁 Папка  reports/            #  для презентаційних матеріалів
│   └── figures/        # Сюди зберігаємо красиві графіки для фінального виступу


📁 Папка src/ (Source Code — чистий код на Python)
Коли в блокнотах (notebooks) ви знайдете робоче рішення, код переноситься сюди у вигляді чистих функцій у файли .py. Це потрібно, щоб блокноти не перетворювалися на тисячі рядків хаосу.

__init__.py — пустий файл. Він просто каже Python, що папка src — це пакет, і з неї можна імпортувати функції в блокноти (наприклад, написати from src.metrics import calculate_overshoot).


config.py       #  константи, налаштування Wi-Fi/портів, шляхи до файлів


data_loader.py (Парсер даних):

Що всередині: Одна або дві функції на Python за допомогою pandas. Наприклад, def load_and_clean_log(file_path):. Ця функція відкриває CSV-файл, прибирає можливі помилки зв'язку (пропущені рядки) і повертає чистий DataFrame.

metrics.py (Математика оцінки):

Що всередині: Чисті математичні формули для аналізу графіків. Функції: calculate_overshoot(data), calculate_settling_time(data), get_fitness_score(data, version).

ga_optimizer.py (Генетичний алгоритм):

Що всередині: Налаштування бібліотеки генетичного алгоритму (наприклад, DEAP або scikit-opt). Тут прописані правила: який розмір популяції, з якою ймовірністю відбувається мутація та схрещування PID-коефіцієнтів.

🛠️ Конфігураційні та інформаційні файли (В корені проєкту)
.gitignore (Ігнорування зайвого):

Що всередині:  Оскільки лог-файли можуть важити багато, в цей файл обов'язково треба записати рядок: data/raw/.

 Список інструментів, які ви використовуєте, з їх версіями. Для старту вашого етапу він виглядатиме так:

pandas

plotly
scikit-opt
jupyter
